# Spike — a tree-sitter parser tier (TypeScript)

**Status: complete. See [`GO-NO-GO.md`](./GO-NO-GO.md) for the decision.**

This is the quarantined spike called for by [`docs/backlog-ast-tree-sitter.md`](../../docs/backlog-ast-tree-sitter.md):
prove that a real parse tree can **enrich the existing `{nodes, edges}` schema** with two things the
zero-dependency regex engine structurally cannot produce precisely, then decide go/no-go on evidence.

It is deliberately **isolated** from the repo's zero-dependency root: its own `package.json`, its own
`node_modules` (git-ignored). The root `package.json` stays dependency-free until — and only if — the
maintainer says GO. Nothing here is wired into the engine.

## What it proves

1. **Exact cyclomatic complexity** — McCabe counted from real control-flow nodes, using the *same
   decision definition* as the shipping `scripts/lib/complexity.mjs`, so every exact-vs-regex gap is a
   **precision gap**, not a definitional one.
2. **Dynamic-dispatch call edges** — `this.m()` and typed-receiver `x.m()` resolved to the method,
   which the regex extractor deliberately drops ("a method call `obj.fn()` is NOT wired").
3. **Determinism** — byte-identical graph across repeated runs with a pinned grammar.

All three preserve codeweb's core guarantees: tree-sitter **never executes** the target (static parse),
and a pinned grammar is deterministic. The only philosophy line that moves is "zero runtime
dependencies" — which the maintainer pre-approved trading for real capability.

## Run it

```sh
cd spike/tree-sitter
npm install          # web-tree-sitter (WASM runtime) + @vscode/tree-sitter-wasm (modern grammars)
npm run spike        # exact-vs-regex complexity table + the dispatch edges the regex drops
npm run determinism  # asserts byte-identical output across 3 derivations (exit 0 = PASS)
```

`npm run spike` reuses the **real** `scripts/lib/complexity.mjs` for the regex side — it is a true
apples-to-apples comparison, not a strawman reimplementation.

## Pinned versions (the ABI lesson)

- `web-tree-sitter` **0.26.9** (runtime, ABI 14)
- `@vscode/tree-sitter-wasm` **0.3.1** → `tree-sitter-typescript.wasm` (ABI 14)

The obvious-looking `tree-sitter-wasms@0.1.13` ships grammars built with `tree-sitter-cli@0.20.x`
(ABI 13) and **fails to load** against a modern runtime with a `dylink` error. The grammar `.wasm`
must match the runtime's ABI window — vendor a known-good matched set and pin both. This is the single
most important operational finding from the spike.
