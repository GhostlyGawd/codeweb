# Activation & First-Win Audit — codeweb

**Date:** 2026-07-23 · **Scope:** the first session itself — everything after install. No signup exists; "activation" here means: the first `/codeweb` (or `npx @ghostlygawd/codeweb`) run, the first `report.html` opened, the first MCP answer inside an agent session, the first hook-injected brief. **Method:** the first session was actually run cold, not traced — `node scripts/run.mjs` against real targets (bench/corpus/flask: 1,668 symbols; bench/corpus/axios: 830 symbols; a 2-file repo; a Lua repo; an empty dir; a broken-syntax repo), a scripted MCP stdio session (initialize → no-graph query → `codeweb_map` → first answers), and all three hooks fired against a mapped fixture. All outputs went to a scratch directory; this file is the only write in the repo. **Lane:** the quality of the first session — empty states, guidance, seed data, setup tax, the inspiring next step, recovery from false starts. Install-step mechanics and step-counts are FUNNEL.md's settled territory (its §4 budget is taken as given); persuasion is CRO.md's; everything after the first map exists is RETENTION.md's. This report starts at the moment the user's finger hits Enter on the first run and stops where RETENTION begins.

---

## 0. The first win, named precisely

The brief demands the first win be named, per persona. They are different moments:

**Human at a terminal (npx / clone / plugin-`/codeweb`):** the first win is **the consolidation advisory printing against your own repo, ~2.3 seconds after Enter** — concretely the block `14 actionable findings · 6 ready · 0 blocked · 8 review / if all ready merges applied: -6 duplication finding(s), ~85 LOC reclaimed (gate would stay green)` followed by named, file:line-cited, safe-to-apply merges (measured on flask; axios 2.2 s; both wall-clock this machine, Node 22.22.2). This is the win *as shipped* — not the interactive map. The map (`report.html`) is the intended wow and the shareable proof, but it sits one manual file-open away (`run.mjs:181` prints the hint; auto-open is opt-in), and in headless/remote sessions — exactly where coding agents run — it is unreachable entirely (FUNNEL §2 owns the serve fix). The terminal advisory needs zero extra steps, lands first, and is specific to *your* code with evidence ("Keep `src/flask/app.py:Flask.send_static_file` · removes 1 copy · rewires 0 callers · blast 0 · ~21 LOC"). Activation design should treat the terminal summary as the first win and the map as the second beat that converts the win into belief.

**Agent-driven session (MCP):** the first win is **the first exact structural answer inside the work loop** — `codeweb_callers` returning `{"summary":"1 callers of add","results":["index.js:<module>"]}` in 1–3 ms, after the server self-healed from nothing: measured cold, the no-graph error returns in 1 ms naming `codeweb_map` (`mcp-server.mjs:98`), `codeweb_map` builds the graph in 619 ms with per-stage progress notifications, and the retried question answers in 3 ms. For this persona "seeing the architecture mapped" is not the win at all — the agent never looks at pictures; the win is an answer grep cannot give (call edges, not text hits), delivered without the agent leaving its task. This path is already at its achievable minimum and is the best-engineered first session in the product.

**Plugin-ambient (hooks):** the first win arrives one session *later* — the ~2 KB brief (`[codeweb] this repo is mapped … load-bearing: render_template×12 … ask codeweb before guessing`) injected at SessionStart, verified firing at sub-100 ms from the sidecar. It only exists after a first `/codeweb`, which nothing after install prompts (FUNNEL §2 habit row — not recounted here; the first-session consequence is in Walk step 6).

## 1. First-session walk (screen by screen)

A CLI product's "screens" are terminal moments and artifacts. Walked cold, in order, with findings inline. `[A#]` references §3.

**Screen 1 — the command.** From the README a newcomer types the npx form with their project path substituted twice, or (plugin) `/codeweb`. Counted and settled in FUNNEL §4; the session-side note is that a *bare* `node run.mjs` / `npx …codeweb` prints usage and exits 2 — a clean teach, except the usage text itself lies about the default out-dir ("`.live/<slug>` under the plugin root", `run.mjs:30`) while the code uses `.codeweb/runs/<slug>` (`run.mjs:9,70`). First contact with the product's voice contains a known-false sentence [A7]. On Node 20 LTS the command dies with a raw syntax stack trace — no version guard exists in `run.mjs` (grep: no `process.version` anywhere in it) despite `engines: >=22` (`package.json:37`) [A6].

**Screen 2 — the pipeline stream (seconds 0–2.3).** Five stages print with wall-times — genuine progress reassurance (`run.mjs:77,89`). But the newcomer's first ten seconds contain developer debug output presented as product output: `cluster` prints `domains: 8  (was 32, hub-named)` and `--- top 18 domains ---` — **both hardcoded strings** (`cluster3.mjs:110,112`), printed identically on a 2-symbol repo ("domains: 1 (was 32…)") and, on an empty map, `isolated-after-dehub: 0 (NaN%)` — a literal NaN. "was 32" is a fossil of some original development run; on a first run there was no "was." A tool whose pitch is determinism and receipts opens by printing two false numbers and a NaN [A1]. `extract`'s line is honest but reads as internals ("dropped 171 ambiguous bare-call edges (9 short-name); wired 72 dispatch edge(s)"); `overlap` prints `findings 14 (incl. 8 drifted) · unverified 3 · dismissed 9` with no glossary for drifted/unverified/dismissed [A5].

**Screen 3 — the advisory (the first win).** `optimize` prints its summary headline — then immediately buries it under the *full* advisory: all 14 finding blocks, ~90 lines, scrolling the headline and the ready tier off-screen before the done banner arrives (observed: the flask first run's total output is ~120 lines, over half of it the optimize dump that is also written verbatim to `optimize.md`) [A2]. The content itself is the best first-win material in the product: tiered ready/blocked/review, "gate: would pass (exit 0)", named keep-symbol, LOC reclaimed.

**Screen 4 — the done banner.** Artifact list + "open …report.html in your browser (or re-run with --open)" (`run.mjs:179-181`) — one clear next step, good. Three gaps: (a) no OS-appropriate open command and no auto-open on TTY (FUNNEL owns); (b) **no bridge to the living-map loop** — the banner never mentions that an agent can now query this map (`claude mcp add …`), that hooks will brief future sessions, or that `report.md` exists as the no-browser fallback for SSH/devcontainer users, so for an npx human the product ends at "done" exactly where its thesis ("living map") should begin [A3]; (c) the value receipt (`codeweb here so far: …`, `run.mjs:184-188`) correctly stays silent on run 1 — but nothing else celebrates or summarizes the win either; the banner is logistics, not a result ("6 ready merges, ~85 LOC reclaimable" never reappears) [A2].

**Screen 5 — report.html first paint.** Verified in the generated artifact (556 KB, self-contained, flask): default tab **Findings** with sections "Consolidation findings (19) — body-confirmed by the overlap pipeline", Duplication, Interface patterns ("NOT copy-paste; don't merge" — excellent pre-emptive teaching, `report-template.html:507`), Hotspots, Likely dead code; the 340 px detail panel first-paints an *overview* — largest areas, "worth a look" counts, and a "how to read this" paragraph (`report-template.html:379-407`) — not an empty pane; empty findings render "No pipeline findings 🎉" and dismissed candidates render as a transparency line (`:503-504`). Graph tab carries an in-canvas legend ("● area — size = symbols, click to expand it · scroll zooms", `:172`). Editor deep-links defer their one config ask (project root) to the moment of first click, stored client-side (`:372-394`) — the model example of deferred setup tax. Two dings: the masthead's fourth number is `N dup-groups` while the terminal said "14 actionable", the tab says 19, and build-report printed "28 overlaps" — four vocabularies for the headline number inside one first session [A5]; and the report never states its own build date (RETENTION R9 owns it; it also reads as missing reassurance in session one).

**Screen 6 — the next session (hooks).** Fired against the mapped fixture: SessionStart brief renders areas/load-bearing/entry-points/tests/known-issues and ends with the perfect agent next step — `ask codeweb before guessing: codeweb_find "<concept>" → codeweb_explain <id> → codeweb_context <id>`. PreToolUse on `helpers.py` returned in ~10 ms: "24 symbol(s), 62 in-repo dependent edge(s) (most depended-on: url_for ×9)" plus the url_for card with top callers and tests — ambient, accurate, advisory. PostToolUse stays silent when nothing regressed — a channel that only speaks when it matters. This loop is genuinely excellent — **but it is unreachable in session one** (brief exists only after a map; in an unmapped repo all hooks exit silently), so the plugin's first session shows zero evidence the plugin exists (FUNNEL's nudge fix; taken as given).

**Screen 7 — the agent-guided `/codeweb` variant.** `commands/codeweb.md:49-60` and `SKILL.md:84-100` route the agent to the same `run.mjs` fast path and script the ending well: "finish by reporting the artifact paths and the top consolidation opportunities — lead with the **ready** tier… then call out anything **blocked**." The human's first `/codeweb` therefore ends with a curated win, not a raw dump — the agent absorbs Screen 2–3's noise. Two frays: `SKILL.md:50-52,158-163` instructs handoffs to `repo-scan`, `refactor-cleaner`, `codebase-onboarding`, `code-tour` — none of which ship in this plugin (`agents/` contains only dissector + domain-mapper; flagged in docs/product-review-2026-07-18.md §"Doc drift", still live), so a diligent agent's "inspiring next step" can be a hunt for skills that don't exist [A8]; and the skill's agent-read fallback for non-native languages is real but the *engine* never mentions it (next screen).

**Screen 8 — the false starts (empty states).** Run cold, verbatim:
- **Wrong/empty dir:** `[extract] no supported source files under <path>` + the extension list + "is this the right directory? Pass --allow-empty to proceed with an empty map." — names what it looked for, questions the premise, offers an escape. A teaching empty state; the best terminal message in the product.
- **Unsupported language (Lua repo):** same message. The escape it offers is a trap and the escape that exists goes unmentioned: the agent-read fallback (`SKILL.md:98-100`) — the actual answer for C/C++/Lua users in agent sessions — appears nowhere in the terminal path [A4].
- **`--allow-empty` taken:** writes a real workspace (77 KB report of nothing) — and now **every future session opens with a confident lie**: the brief announces `[codeweb] this repo is mapped … 0 symbols / 0 files / 0 domains (unknown languages) … ask codeweb before guessing: codeweb_find …` (verified live). The false start is not recovered from; it is *institutionalized* — the agent is instructed to consult an oracle that knows nothing, in perpetuity [A4].
- **Broken syntax:** a mangled `.py` extracts via regex fallback without crashing or complaining — resilient, though silently degraded (no "1 file parsed degraded" note; acceptable).
- **Re-run:** second identical run reuses stages ("stages reused (fragment unchanged) — skipping downstream recompute; --full forces") and the receipt line appears once stats exist — safe, idempotent, and the one place run-over-run progress shows. (That the re-run is otherwise a repeated first run is RETENTION R1.)
- **MCP cold (no graph):** 1 ms actionable error naming the fix; `codeweb_map` needs zero arguments; progress notifications stream; retry answers in 3 ms. The gold standard the other paths should be graded against.

## 2. Time-to-first-win — today vs achievable minimum

Finger-step counts are FUNNEL §4's settled numbers; this table adds the measured *session* time and where the win actually lands.

| Persona / path | Today | Win lands at | Achievable minimum |
|---|---|---|---|
| Human, npx | 1 long command (path typed 2×) → **2.2–2.3 s pipeline** (+ cold npx fetch) → advisory in terminal; map +1 manual file-open (impossible headless) | Terminal advisory, <1 min total if the command was right | Same wait, 1 bare command from the project dir with auto-open on TTY (FUNNEL fix #2); **win itself is already at minimum — the risk is it scrolling past unread [A1/A2], not arriving late** |
| Human, plugin `/codeweb` | Install steps (settled) + restart + `/codeweb`; agent relays ready tier | Agent's summary message, ~1 turn after `/codeweb` | Already good once `/codeweb` is typed; the gap is that nothing prompts that first `/codeweb` (FUNNEL) and the skill's phantom handoffs fray the ending [A8] |
| Agent, MCP | 1 config command once; first structural question self-heals (1 ms error → 619 ms map → 3 ms answer, measured) | First tool answer, zero marginal steps | **At minimum now.** Protect with tests; copy its pattern elsewhere |
| Human, unsupported language | Dead end with a trap escape; agent fallback exists but is unadvertised | Never (terminal path) | One sentence in the empty-state message routes to `/codeweb` agent fallback; empty maps marked as empty [A4] |

The pipeline is not the tax anywhere — 2.3 s to a real advisory on a real repo is exceptional. Every activation loss is in *presentation and routing*: noise inside the win, a buried headline, a missing bridge to step two, and one poisoned escape hatch.

## 3. Findings

Ranked by distance to the first win (closest first). Effort: S = hours–day · M = days.

### A1 · The first 10 seconds print hardcoded false numbers and a NaN
- **Lens:** 7 (progress & reassurance) · **Location:** `cluster3.mjs:110` (`domains: ${n}  (was 32, hub-named)` — "was 32" is a string literal, printed on every repo including a 2-symbol one), `cluster3.mjs:112` (`--- top 18 domains ---` regardless of count), empty-map run prints `isolated-after-dehub: 0 (NaN%)`; `extract`/`overlap` lines dense with internals ("dropped 171 ambiguous bare-call edges (9 short-name)").
- **Newcomer feels:** "was 32? It's my first run. top *18*? It listed 8. NaN%? — do I trust the other numbers?" This lands *inside* the delivery of the first win, on a product whose entire pitch is that its numbers are receipts.
- **Fix:** delete the two fossil strings (`(was 32, hub-named)` → nothing or the real pre-dehub count the code computes; `top 18` → `top ${shown}`); guard the NaN (0-file case → omit the line); one editorial pass over stage prints: keep counts, move diagnostics behind a `CODEWEB_VERBOSE=1` style lever. · **Effort: S** (the two literals are minutes).

### A2 · The advisory buries its own headline; the done banner is logistics, not a result
- **Lens:** 2 (time-to-first-win) + 7 · **Location:** `optimize.mjs` prints the full 14-block advisory to the terminal (~90 of the first run's ~120 lines) *after* its 2-line headline, duplicating `optimize.md` verbatim; the done banner (`run.mjs:175-188`) then lists six file paths and never restates the result. The win's headline — "6 ready merges, ~85 LOC reclaimable, gate stays green" — is off-screen by the time the run ends.
- **Newcomer feels:** a wall of scrollback; the moment of "it found something real in MY code" is something they must scroll *up* to reconstruct. In an agent session this also burns ~90 lines of context per map for content already on disk.
- **Fix:** terminal shows headline + top 3 ready merges + "full advisory: optimize.md"; the done banner leads with the result line ("mapped 1,668 symbols → 14 findings, 6 ready merges (~85 LOC) — details: optimize.md") before the paths. Full dump stays available behind a flag. · **Effort: S**.

### A3 · The first win ends at "done" — no bridge to the living map (the second step)
- **Lens:** 6 (inspiring next step) · **Location:** `run.mjs:175-188` — after first success the banner offers exactly one action (open report.html). Nothing mentions: the MCP server that would let the user's agent query the map they just built (`claude mcp add codeweb …`, README.md:171), the hooks that will brief future sessions, `report.md` as the no-browser fallback, or re-running `/codeweb` after edits. The receipt line correctly can't speak on run 1 (`stats` empty), so run 1 ends with zero forward pull.
- **Newcomer feels:** "neat. *[closes terminal]*" — a one-shot analyzer impression of a product whose differentiator is the ambient loop. The npx human — the largest cold audience — is never told the product has a second act; momentum dies at the exact peak of conviction (CRO C3's dead-end pattern, inside the product's own terminal).
- **Fix:** a 3-line `next:` block in the first-run done banner (suppressed once stats show activity, so it self-retires): `open <ws>/report.html` (or `report.md` if no display) · `your agent can query this map: claude mcp add codeweb -- npx -y -p @ghostlygawd/codeweb codeweb-mcp` · `after edits: re-run to see the delta`. Tees directly into RETENTION's "since last map" hook — the same banner becomes the delta banner on run 2+. · **Effort: S**.

### A4 · `--allow-empty` converts a clean failure into a permanent confident lie
- **Lens:** 8 (recover from a false start) + 1 (empty states) · **Location:** escape offered at the empty-target message (`extract` guard; `run.mjs:33,46`); resulting state verified: brief opens every later session with `[codeweb] this repo is mapped … 0 symbols / 0 files / 0 domains (unknown languages) … ask codeweb before guessing: codeweb_find …` (`session-brief.mjs:44` + `brief-core`); MCP serves the empty graph without comment; the terminal empty-state never mentions the agent-read fallback that actually handles non-native languages (`SKILL.md:98-100`), and `NO_GRAPH`/brief have no "this map is empty" variant. (RETENTION R11a covers the *repeat* failure memory; this is the poisoned-success state.)
- **Newcomer feels:** their Lua/C++ repo either dead-ends, or — following the tool's own suggestion — enters a state where every future session is told codeweb knows this repo (it knows nothing) and to consult it before guessing. The agent obeys, gets empty answers, and codeweb's reputation in that repo is "confidently useless."
- **Fix:** (a) 0-symbol graph ⇒ brief says `[codeweb] map exists but is EMPTY (no supported languages) — the /codeweb agent fallback can map this repo by reading` and MCP annotates answers likewise; (b) the empty-target message adds one routing line: "Non-native language? In Claude Code, `/codeweb` falls back to agent-based mapping."; (c) demote `--allow-empty` in the message to its real use (monorepo corner cases), since for a newcomer it is almost always the wrong door. · **Effort: S**.

### A5 · Four vocabularies for the headline number inside one session
- **Lens:** 7 (reassurance) · **Location:** same flask run, same graph: overlap stage prints `findings 14 … unverified 3 · dismissed 9`; optimize prints `14 actionable`; build-report prints `28 overlaps`; report masthead computes `N dup-groups` (`report-template.html:328`) while its Findings tab shows `Consolidation findings (19)` (`ovLive` = 28 − 9 refuted, `:452-457`). Graph truth: 28 entries = 6 high + 8 medium + 5 low + 9 refuted.
- **Newcomer feels:** "is it 14, 19, or 28?" at the precise moment they are deciding whether this tool's numbers can be trusted — the in-product sibling of CRO C2's 20-vs-27 finding.
- **Fix:** one shared vocabulary rendered from one helper: **actionable (14) · needs-review/low (5) · dismissed (9, shown as transparency)**; every surface prints the same triple; extend `check-consistency.mjs`'s philosophy to these runtime strings via a unit test over a fixture graph. · **Effort: S–M**.

### A6 · Node 20 LTS: the first command crashes with a raw stack trace
- **Lens:** 5 (setup tax) + 8 (false starts) · **Location:** no `process.version` guard in `run.mjs` (or `mcp-server.mjs`); `engines >=22` is advisory (`package.json:37`); published tarball even declares `>=20` (CRO C5). First noted RETENTION R11b; it belongs to this lane because it is the first five seconds of the first session for a large share of the audience.
- **Newcomer feels:** an unexplained `SyntaxError` from a tool promising determinism; indistinguishable from "broken product."
- **Fix:** two-line guard at the top of both bins: `const [maj] = process.versions.node.split('.'); if (+maj < 22) { console.error('codeweb needs Node >= 22 (you have ' + process.version + ')'); process.exit(1); }`. · **Effort: S** (minutes).

### A7 · The usage text teaches a wrong default on first contact
- **Lens:** 3 (guided first action) · **Location:** `run.mjs:30` usage says default out-dir is `.live/<slug>`; code uses `.codeweb/runs/<slug>` (`run.mjs:9,70`). FUNNEL flagged it inside install friction; it is also the literal first screen of the bare-command path.
- **Newcomer feels:** follows the usage to find their artifacts; the directory doesn't exist.
- **Fix:** correct the string (or, per FUNNEL #2, make the default `<SRC>/.codeweb` and say so). · **Effort: S** (one line; FUNNEL's defaults fix supersedes it).

### A8 · The skill's "next steps" point at four plugins that don't ship
- **Lens:** 6 (inspiring next step) + 3 · **Location:** `SKILL.md:50-52` ("Reuse, don't reinvent… hand off to `repo-scan`… `codebase-onboarding`… `refactor-cleaner`") and `SKILL.md:158-163` (Handoffs: + `code-tour`); `agents/` ships only `codeweb-dissector` and `codeweb-domain-mapper`; already flagged in docs/product-review-2026-07-18.md, still live.
- **Newcomer feels:** (via their agent) the first session ends with the agent searching for, or apologizing about, skills that don't exist — or worse, silently skipping the handoff and the "act on the list" step with it.
- **Fix:** mark each as "if installed, else: [inline 1-line alternative]" — e.g. refactor-cleaner → "apply the top ready merge from optimize.md and re-run /codeweb to verify the delta" (which *is* the ideal second step). · **Effort: S**.

### A9 · Seed data exists but only outside the product
- **Lens:** 4 (seed & sample data) · **Location:** the axios live demo (README.md:55-56, site) is the product's sample data — one click, zero install, genuinely excellent. Inside the terminal product there is no equivalent: no `try it on a sample` path, and the clone path never mentions `bench/corpus/{axios,flask}` sitting right there in the repo. The report/demo tab-default question is CRO C10's (not re-argued).
- **Newcomer feels:** (only for the no-browser or straight-to-terminal cohort) the first output they ever see is on their *own* repo — high stakes if their repo is huge, weird, or empty. Low severity: mapping your own repo in 2.3 s largely obsoletes sample-first onboarding.
- **Fix:** one line in the clone path of README ("kick the tires: `node scripts/run.mjs bench/corpus/flask --out-dir /tmp/flask-map`"); nothing heavier. · **Effort: S**.

**What is already right (protect, don't touch):** the 1 ms `NO_GRAPH` error that names its own fix and the 619 ms self-heal with progress (`mcp-server.mjs:98,252-253`); the empty-target message that questions the premise; unknown flags rejected with usage (`lib/cli.mjs` parseArgs); the report's teaching first paint (overview panel + "how to read this" + "No pipeline findings 🎉" + the Interface-patterns "don't merge" warning); the lazily-asked editor root; idempotent re-runs with honest "stages reused"; the receipt that never speaks before it has something to say; hooks that stay silent rather than guess. These are the activation patterns the fixes above should copy, and several are test-pinned — keep them that way.

## 4. The one change

**Rewrite the first-run terminal ending into a "first-win banner": clean stages → headline result → top-3 ready merges → artifacts → a 3-line `next:` block — and make the same banner render the since-last-map delta on run 2+.** (A1 + A2 + A3 as one S–M unit of work in `run.mjs`/`cluster3.mjs`/`optimize.mjs` terminal output only; no engine changes.)

Why this one: the pipeline already delivers the first win in 2.3 seconds — faster than any realistic onboarding could promise — and then walks it past the user inside 120 lines of scrollback containing hardcoded false numbers, with no second act. Every persona crosses this exact surface: the npx human reads it raw, the `/codeweb` human gets it relayed by an agent whose summary quality depends on what the tail says, and the MCP agent pays its length in context tokens. Fixing the banner is the cheapest unit that (a) removes the trust-damaging noise from the win's delivery, (b) puts the headline where the session ends instead of mid-scroll, and (c) gives the win a next step so activation doesn't terminate at "done". The install-side one-change (FUNNEL's marketplace.json + npx defaults) gets more people *to* this screen; this is the highest-leverage change *of* the screen, and the two compound.

**The second step it tees up:** the `next:` block's last line — "after edits: re-run `/codeweb` to see what changed" — is a promise the banner itself keeps once RETENTION's #1 hook ("since last map" delta + `history.jsonl`) lands in the same print site: run 1 ends with *next steps*, run 2+ ends with *progress* ("dups 12→9 since Tuesday"). One surface, first-win then habit — activation handing off to retention in the same lines of code.
