Mapping axios: 3 confirmed duplications in a 50M-download library

We pointed codeweb at axios — the HTTP client downloaded ~50M times a week — read-only, no setup, one command. Measured at axios v1.18.1 (a209bfb1); the pinned command below reproduces these numbers exactly.

The unedited generated report — click around it live.

The map, in one command

node scripts/run.mjs path/to/axios/lib --target axios
# 274 symbols · 374 edges · 8 domains · 17 overlap candidates

Domains auto-clustered from the code's own structure: helpers, core, adapters, cancel, defaults, platform/node, platform/common, root.

Precision, not noise: 17 candidates → 3 real duplications

Anyone can grep for same-named functions. codeweb confirms each overlap against the actual function bodies (token-shingle similarity), so the 17 raw candidates resolve honestly:

3 body-confirmed

Genuinely the same logic in two places — two of them byte-for-byte identical across files.

12 dismissed

Same name, different bodies. Refuted so you don't chase them — that 12-of-17 dismissal rate is the point.

2 unverified

Left for a human to judge, not asserted.

And the safety call that a linter can't make

codeweb simulates each merge against its own regression gate: two of the three merges would introduce a new dependency cycle (verdict: blocked, with the fix — host the canonical in a neutral module). The safe one (setFormDataHeaders, 91% body match) is marked ready: the gate stays green and the duplication count drops by one.
